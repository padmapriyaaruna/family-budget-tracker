ARTWORK AUTOMATION PROCESS DOCUMENT 
For every order which is processed by Sainmarks/Britannia, there is a process wherein 
Artwork of the end product is generated and sent for approval to the client. Once client 
approves, the product is taken up for production. 
Overall, there are multiple ways in which orders are processed. 
ΓÇó Orders are placed in BGP Connect, which are then processed using a 
combination of ESCO and XMPie using an automatic work∩¼éow and the resultant 
artwork is sent to client for approval. 
ΓÇó Orders are placed in BGP Connect. Upon submission, there is an internal 
subroutine referred to a ΓÇ£BratΓÇ¥, which generates an XML File. This ∩¼üle is then 
converted into a CSV and is uploaded along with the respective Template to 
XMPie which generates the artwork variations. This is then re-purposed manually 
into the Sainmarks/Britannia format which is then manually emailed to the client 
for approval. 
ΓÇó For Care Labels, there is another software being used known as NICE Label. 
Input ∩¼üles for care labels are uploaded into a Secure FTP Server in XML format. 
These ∩¼üles are automatically picked up from this Secure FTP Server by NICE 
Label and sent directly to the printer after applying the appropriate design 
template. 
All of the above software are commercial software which involve large licensing costs. 
One of the primary objectives that were behind the start of this project to build our own 
Artwork Generation Engine was to try and reduce the recurring licensing costs which are 
currently being incurred. 
At present we have an engine which we have developed using Python, Latex and 
GhostScript which is able to take the XML ∩¼üle generated by BGP Connect as an input 
and using a SVG Template, generate the required labels. In addition, we have also built a 
web interface for a ΓÇ£CustomerΓÇ¥ to place an order and get the artwork generated.

Step 1 ΓÇö Prepare Your InDesign SVG Template 
When exporting SVG from InDesign, you need to mark variable fields so your engine can 
find and replace them. Two strategies: 
Strategy A ΓÇö Placeholder Text (Simplest) 
Replace variable text in InDesign before export with tokens: 
{{product_name}} 
{{sku}} 
{{net_weight}} 
{{batch_no}} 
The SVG will contain these literally inside <text> or <tspan> elements. 
Strategy B ΓÇö Element IDs (More Robust) 
In InDesign, name your layers/objects. These become id attributes in SVG: 
<text id="product_name" ...>Sample Product</text> 
<text id="sku" ...>000000</text> 
<image id="product_image" .../> 
Your engine targets elements by ID and replaces content ΓÇö cleaner and doesn't require 
template re-export when placeholder names change. 
Recommended: Use both ΓÇö IDs for targeting, placeholder text as fallback.

import lxml.etree as ET 
 
def load_from_xml(xml_path: str) -> list[dict]: 
    tree = ET.parse(xml_path) 
    root = tree.getroot() 
    records = [] 
    for item in root.findall("item"):  # adjust tag to your XML schema 
        records.append({ 
            "product_name": item.findtext("product_name"), 
            "sku":          item.findtext("sku"), 
            "net_weight":   item.findtext("net_weight"), 
            "batch_no":     item.findtext("batch_no"), 
        }) 
    return records

Step 3 ΓÇö SVG Template Engine 
import lxml.etree as ET 
import copy 
 
SVG_NS = "http://www.w3.org/2000/svg" 
XLINK_NS = "http://www.w3.org/1999/xlink" 
 
def inject_data(svg_path: str, data: dict) -> ET._Element: 
    tree = ET.parse(svg_path) 
    root = tree.getroot() 
 
    # --- Strategy A: Replace {{placeholder}} tokens in all text nodes --- 
    for element in root.iter(): 
        if element.text: 
            for key, value in data.items(): 
                element.text = element.text.replace(f"{{{{{key}}}}}", 
str(value)) 
        if element.tail: 
            for key, value in data.items(): 
                element.tail = element.tail.replace(f"{{{{{key}}}}}", 
str(value)) 
 
    # --- Strategy B: Target elements by ID --- 
    for key, value in data.items(): 
        el = root.find(f'.//*[@id="{key}"]') 
        if el is not None: 
            # Handle <text> and <tspan> elements 
            if el.tag in [f"{{{SVG_NS}}}text", f"{{{SVG_NS}}}tspan", 
"text", "tspan"]: 
                el.text = str(value) 
            # Handle <image> elements (swap image href) 
            elif el.tag in [f"{{{SVG_NS}}}image", "image"]: 
                el.set(f"{{{XLINK_NS}}}href", str(value)) 
 
    return root 
 
Step 4 ΓÇö Render SVG ΓåÆ Output 
import cairosvg 
import os 
 
def render_label(svg_root: ET._Element, output_path: str,  
                 format: str = "pdf", dpi: int = 300): 
    svg_bytes = ET.tostring(svg_root, xml_declaration=True, encoding="UTF-
8") 
     
    os.makedirs(os.path.dirname(output_path), exist_ok=True) 
 
    if format == "pdf": 
        cairosvg.svg2pdf(bytestring=svg_bytes, write_to=output_path, 
dpi=dpi) 
    elif format == "png": 
        cairosvg.svg2png(bytestring=svg_bytes, write_to=output_path, 
dpi=dpi) 
    elif format == "tiff": 
        # CairoSVG ΓåÆ PNG ΓåÆ TIFF via Pillow 
        from io import BytesIO 
        from PIL import Image 
        png_bytes = cairosvg.svg2png(bytestring=svg_bytes, dpi=dpi) 
        img = Image.open(BytesIO(png_bytes)) 
        img.save(output_path, format="TIFF", compression="tiff_lzw") 
 
Step 5 ΓÇö Batch Runner 
def batch_render(template_path: str, records: list[dict],  
                 output_dir: str, format: str = "pdf"): 
    print(f"Rendering {len(records)} labels...") 
    for i, record in enumerate(records): 
        svg_root = inject_data(template_path, record) 
        filename = f"{record.get('sku', f'label_{i}')}.{format}" 
        output_path = os.path.join(output_dir, filename) 
        render_label(svg_root, output_path, format=format) 
        print(f"  [{i+1}/{len(records)}] ΓåÆ {filename}") 
    print("Done.")

What You Gain with Inkscape 
Direct ID assignment ΓÇö in Inkscape's XML editor (Ctrl+Shift+X), you can set 
id="Order_number" on any element directly. No workarounds needed. 
Native SVG gradients ΓÇö design gradients in Inkscape, they export as standard 
<linearGradient> / <radialGradient> elements that cairosvg renders perfectly. 
Clean group structure ΓÇö create layers in Inkscape, they become <g id="layer_name"> 
elements ΓÇö exactly what your layout variant switching logic already uses. 
Live text ΓÇö all text stays as <text> and <tspan> elements, injectable by your engine 
without any font-outlining issues.
